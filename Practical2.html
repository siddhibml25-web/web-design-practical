What are Semantic Elements?

HTML5 introduced semantic elements that clearly describe their meaning to both the browser and the developer. Instead of using generic <div> tags everywhere, we now use meaningful tags like <header>, <nav>, <main>, <section>, <article>, <aside>, and <footer>.

Why Semantic HTML Matters

Semantic HTML improves accessibility, helps search engines better index content, and makes code more readable and maintainable.

New HTML5 Semantic Tags

<header> — Introductory content or navigation.
<nav> — A set of navigation links.
<main> — The main content of the document.
<section> — A standalone section of content.
<article> — Self-contained content.
<aside> — Related side content.
<footer> — Footer of the page or section.
